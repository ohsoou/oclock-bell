# 정시 알람 (O'Clock Bell)

설정한 시간 범위 안에서 정시마다 알림을 보내고, 앱이 열려 있을 때는 한국어 TTS로 시간을 읽어주는 PWA 기반 알람 앱입니다. 웹앱으로 동작하면서도 Android WebView 네이티브 래퍼 환경을 함께 지원합니다.

## 현재 상태

현재 버전: `2.1.0`

| 항목 | 상태 |
|------|------|
| 실시간 시계 | 시/분/초 1초 단위 갱신 |
| 시간 보정 | Worker에서 NTP API를 사용해 시계 오프셋 보정 시도 |
| 알람 범위 | 시작/종료 시각 설정, 자정 넘김 범위 지원 |
| 테스트 모드 | 1시간 간격 대신 1분 간격 알람 |
| 음성 안내 | 한국어 TTS, 음높이/속도/음량 조절, 테스트 재생 |
| 설정 가이드 | 홈/설정 화면 온보딩 오버레이 제공 |
| 백그라운드 알림 | Service Worker + Notification + 환경별 TimestampTrigger 지원 |
| Android 래퍼 지원 | 정확한 알람/배터리 최적화/네이티브 상태 연동 지원 |
| 버전 표시 | 설정 화면 하단에 현재 버전 노출 |

## 주요 기능

- 정시 알람: 설정한 시간 범위 안에서 매 정시마다 알람을 시도합니다.
- 한국어 TTS: 앱이 열려 있을 때 현재 시각을 한국어 음성으로 읽어줍니다.
- 다음 알람 안내: 메인 화면에서 다음 알람 예정 시각을 보여줍니다.
- 위치 기반 테마: 위치 정보가 있으면 일출/일몰 기준으로 시계 테마를 전환합니다.
- 상태 알림: 웹 환경에서는 알림창에 알람 켜짐/꺼짐 상태를 상시 표시할 수 있습니다.
- 테스트 모드: 정시 대신 1분 단위로 빠르게 알람 흐름을 확인할 수 있습니다.
- 설정 온보딩: 홈과 설정 화면의 핵심 UI를 순서대로 안내합니다.
- 버전 관리: `version.js`에서 앱 버전을 관리하고, 설정 화면 하단에 표시합니다.

## 플랫폼 동작

### 웹앱(PWA)

- Service Worker로 앱 셸을 캐시합니다.
- Web Worker가 시계 틱과 정시 경계 계산을 담당합니다.
- Notification 권한이 허용되면 백그라운드 시스템 알림을 예약합니다.
- `TimestampTrigger`를 지원하는 환경에서는 OS 레벨 예약 알림을 사용합니다.
- Wake Lock과 Silent AudioContext로 백그라운드 중단 완화를 시도합니다.

### Android WebView 래퍼

- `NativeAlarm` 브리지가 있으면 네이티브 상태를 우선 사용합니다.
- 정확한 알람 권한, 배터리 최적화 예외 상태를 설정 화면에 표시합니다.
- 배터리 최적화 예외 요청 버튼을 제공합니다.
- 네이티브 모드에서는 웹 전용 상태 알림/음성 선택 UI를 감춥니다.

## 설정 화면

### 알림

- 앱 알림 권한 상태 표시
- 백그라운드 지원 여부 표시
- 웹 환경용 상태 알림 토글
- Android 래퍼용 정확한 알람/배터리 최적화 상태 표시

### 목소리 / 음성 조절

| 항목 | 범위 | 기본값 |
|------|------|--------|
| 목소리 | 시스템 음성 목록 | 자동 (한국어 기본) |
| 음높이 | 0.5 ~ 2.0 | 1.5 |
| 말하기 속도 | 0.5 ~ 1.5 | 0.8 |
| 음량 | 0.0 ~ 1.0 | 1.0 |

### 개발 / 테스트

- 테스트 모드: 정시 대신 1분 간격
- 목소리 테스트: 현재 설정으로 즉시 발화
- 기본값으로 초기화
- 설정 하단 버전 표시

## 아키텍처

```text
index.html
  - 메인/설정 화면 마크업
  - 가이드 오버레이와 설정 UI

app.js
  - 시계 UI 렌더링
  - 설정 저장 및 상태 표시
  - TTS 재생, 권한 처리, 페이지 전환
  - 웹/네이티브 래퍼 분기 처리

timer.worker.js
  - NTP 동기화
  - 1초 틱 생성
  - 다음 정시/정분 경계 정밀 예약

sw.js
  - 앱 셸 캐시
  - 백그라운드 알림 예약/취소
  - 상태 알림 표시

version.js
  - 앱 버전 및 릴리즈 날짜 관리
```

## 로컬 실행

Service Worker는 `file://`에서 동작하지 않으므로 로컬 서버가 필요합니다.

```bash
npx serve "oclock bell"
```

또는

```bash
python3 -m http.server 8080 --directory "oclock bell"
```

브라우저에서 `http://localhost:3000` 또는 `http://localhost:8080`으로 접속해 확인합니다.

## 제약 사항

- 웹앱은 기기의 무음/진동 모드 여부를 직접 감지할 수 없습니다.
- 웹 환경에서는 앱이 완전히 종료된 뒤 TTS 재생을 보장할 수 없습니다.
- 브라우저와 OS 정책에 따라 백그라운드 예약 알림의 신뢰도는 달라집니다.
- 완전한 종료 상태 보장이 필요하면 Push 서버 또는 네이티브 앱 지원이 필요합니다.
